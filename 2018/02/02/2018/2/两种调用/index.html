Java的填坑之路——he函数参数的两种调用

发表于 2018-02-02 | 分类于 Java | 字数统计: 1,272字 | 阅读时长 ≈ 5分钟

值传递与引用传递

程序设计语言中将参数传递给函数方法有两种，，按值传递(call by value)表示方法接收的是调用者提供的值。
而按引用传递(call by reference)表示方法接收的是调用者提供的变量地址。前者不可以修改原参数的值，而
后者可以，因为它修改的是那个地址里边的东西，原参数肯定会变。而Java总是采用按值调用
比如一个方法：

private static void tripleValue(double per) {
        per*=3;
        System.out.println("方法结束时：per=" + per);
}

但是你调用，却不会改变传入的值：

double per = 10;
System.out.println("执行方法之前:per=" + per);
tripleValue(per);
System.out.println("执行方法之后:per=" + per);

方法执行后per的值仍然是10。

然而方法参数共有两种类型：

  • 基本参数类型（数字，布尔值）
  • 对象

一个方法不可能修改一个基本数据类型的参数，但是对象作为参数就不同了，可以利用下边的这个方法实现更改:

private static void tripleSalary(Employ x) {
    x.raiseSalary(120);
    System.out.println("方法结束时：salary=" + x.getSalary());
}

执行下边一条命令：

Employ harry = new Employ("Harry", 50000);
System.out.println("执行方法之前:salary=" + harry.getSalary());
tripleSalary(harry);

结果：

测试对象：
执行方法之前:salary=50000.0
方法结束时：salary=110000.0
执行方法之后:salary=110000.0

方法结束后，harry相应属性的值已经改变。按理说Java是按值传递，不应该改变才对，再看下边一个方法：

private static void swap(Employ x, Employ y) {
   Employ e = x;
   x = y;
   y = e;
   System.out.println("方法完成时:x=" + x.getName());
   System.out.println("方法完成时:y=" + y.getName());
}

/**
 * Test3: 测试对象的传递类型
 */
System.out.println("\n测试对象：");
Employ jack = new Employ("Jack", 50000);
Employ marry = new Employ("Marry", 0);
System.out.println("执行方法之前:Jack = " + jack.getName());
System.out.println("执行方法之前:Marry = " + marry.getName());
swap(jack, marry);
System.out.println("执行方法之后:Jack = " + jack.getName());
System.out.println("执行方法之后:Marry = " + marry.getName());

执行结果：

测试对象：
执行方法之前:Jack = Jack
执行方法之前:Marry = Marry
方法完成时:x=Marry
方法完成时:y=Jack
执行方法之后:Jack = Jack
执行方法之后:Marry = Marry

方法并没有改变储存在jack和marry中对象引用，swap方法的参数只是x和y被初始化为两个对象的引用,
这个方法交换的是两个拷贝。

在这里总结一下：我认为对对象的引用就是对地址的一份copy，而对基本参数的传递就是按值，然而为什
么上边的对象的传递为什么会出现上述结果呢？

比如说你new了一个对象harry，这个对象地址是0x1234，在方法中方法参数x是对harry对象地址的一个copy，地址
也是0x1234，当你在方法中修改对象的状态时，改变的是x所指地址里边状态的改变，你在方法外边访问harry对象时，
地址还是0x1234但是该地址里边的值已经在方法里边改变了，所以harry对象的状态也会改变。

现在你new了两个对象jack和marry，地址分别为0x1235和0x1236,当进入swap方法时，两个参数x和y分别是对jack
和marry地址的copy,x指向地址0x1235,而y指向0x1236，方法里边交换的只是地址值，方法结束时，x指向地址0x1236,
而y指向0x1235,但方法外边jack和marry还是原地址，里边的值当然不会改变了 哈哈哈!

完整代码如下：

public static void main(String[] args) {

    /**
     * Test1: 测试变量的传递类型
     */
    System.out.println("测试变量：");
    double per = 10;
    System.out.println("执行方法之前:per=" + per);
    tripleValue(per);
    System.out.println("执行方法之后:per=" + per);

    /**
     * Test2: 测试对象的传递类型
     */
    System.out.println("\n测试对象：");
    Employ harry = new Employ("Harry", 50000);
    System.out.println("执行方法之前:salary=" + harry.getSalary());
    tripleSalary(harry);
    System.out.println("执行方法之后:salary=" + harry.getSalary());

    /**
     * Test3: 测试对象的传递类型
     */
    System.out.println("\n测试对象：");
    Employ jack = new Employ("Jack", 50000);
    Employ marry = new Employ("Marry", 0);
    System.out.println("执行方法之前:Jack = " + jack.getName());
    System.out.println("执行方法之前:Marry = " + marry.getName());
    swap(jack, marry);
    System.out.println("执行方法之后:Jack = " + jack.getName());
    System.out.println("执行方法之后:Marry = " + marry.getName());

}

private static void swap(Employ x, Employ y) {
    Employ e = x;
    x = y;
    y = e;
    System.out.println("方法完成时:x=" + x.getName());
    System.out.println("方法完成时:y=" + y.getName());
}

private static void tripleSalary(Employ harry) {
    harry.raiseSalary(120);
    System.out.println("方法结束时：salary=" + harry.getSalary());
}

private static void tripleValue(double per) {
    per*=3;
    System.out.println("方法结束时：per=" + per);
}
// 输出：
